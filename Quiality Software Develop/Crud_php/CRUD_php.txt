Descripción.

Desarrollar un programa que permita la creación, modificación, eliminación y consulta (CRUD) de
registros de productos y usuarios, incluir el módulo de autenticacion para el login de usuarios.
Este será una aplicación web con una página home y el menú para acceder a ambos módulos, desde la
perspectiva de usuario administrador.

Requisitos funcionales:
RF01: El sistema debe permitir la autenticación de los usuarios registrados con su usuario y
contraseña. El sistema valida en diferentes casos en los cuáles la autenticación falla y así
arroja mensajes de acuerdo al nivel de error que se presente en la autenticación: usuario no existe,
error en contraseña.
RF02: El sistema permite el registro de la información de los usuarios que se lleva a cabo por el
administrador. La información a tomar en cuenta es: nombre, apellido, domicilio, celular, email,
fecha de nacimiento. El sistema permite validar si la información de los nuevos registros existe
previamente en la base de datos, evitando registros duplicados.
RF03: El sistema permite llevar a cabo las modificación de la información registrada de los usuarios
por el administrador. Se verifica que la información actualizada no duplique un registro creado
previamente.
RF04: El sistema permite al administrador eliminar los registros del usuario que desee.
RF05: El sistema permite llevar a cabo consultas de la información de los usuarios que ha sido
registrada. Las consultas se pueden filtrar por los distintos campos que se manejen en la tabla de
usuarios.
RF06: El sistema permite el registro de la información de productos manejando los datos: nombre,
descripción, precio, marca, imagen.
RF07: El sistema permite la generación de un código QR para asociar la página del producto mismo con
el uso del QR. Datos a considerar para la generación: ID y el URL de la página para el producto
especificado.
RF08: El sistema permite la modificación de la información de los productos por el adminisrador,
excluyendo ID y QR.
RF09: El sistema permite la eliminación del registro de producto que seleccione el administrador en la
página correspondiente.
RF10: El sistema permite las consultas de productos por sus diversos campos en la tabla de la base
de datos. El QR dirige mediante el URL y el parámetro del GET hacia la página de detalle del
producto que se quiere consultar.

Requisitos no funcionales:
RNF01: El sistema despliega una página HOME mediante la petición a la página index.php en el sitio
donde se monte. La página de HOME contiene los elementos: banner, navigation, carrusel de los productos,
información de la empresa, en el navigation tiene las secciones: quienes somos, productos,contacto y
un button de login. Además tiene un footer con mapa del sitio y redes sociales.
RNF02: El sistema abre una página adicional para cada uno de los elementos del navbar, incluyendo
el login para la sección de administrador.
RNF03: La página de quienes somos será html estático que se abre mediante el click en el navbar.
RNF04: Colores.
